### 文章解读：树莓派5驱动ST7796S显示屏（DRM/fbtft）

#### 1. 文章主题与目标

本文是一篇实践性极强的技术教程，主要目标是在**树莓派5**上驱动**ST7796S** SPI接口的TFT液晶屏（4.0寸，分辨率480×320）。文章提供了两种显示框架的适配方案：
- **DRM (Direct Rendering Manager)** — 官方推荐，基于 Linux 内核标准显示框架。
- **fbtft (Framebuffer TFT)** — 一个历史较久的第三方驱动框架，本文也给出了完整方案。

此外还包含触摸屏驱动（ads7846）的配置，但注明“官方优化较差，不推荐使用”。

#### 3. 关键技术点

- **MIPI DBI 固件格式**：`mipi-dbi-cmd` 脚本将文本命令（command/delay）转换为内核 `mipi-dbi-spi` 驱动的二进制固件（`panel.bin`）。该格式魔数为 `MIPI DBI\x00...`，版本1，是 DRM 驱动标准化初始化方式。
- **设备树 overlay**：
  - `mipi-dbi-spi` 使用 `spi0-0`（SPI0，CS0），速度 48MHz，`write-only` 标志（因为 SPI 读取较慢）。
  - `fbtft` 方案使用自定义 `rpi5-st7796s.dts`，需自行编译。
- **内核降级**：树莓派 OS 默认内核（如 6.12.38+）可能不提供 `linux-headers` 包，导致无法编译内核模块。降级到 `6.12.34+rpt-rpi-v8` 并手动拷贝 `vmlinuz` 和 `initrd.img` 是必要步骤。

#### 4. 文章亮点与坑点总结

**亮点：**
- 提供了两种方案，覆盖不同用户需求（DRM 更现代化，fbtft 更传统）。
- 附带了完整的初始化命令序列（command.txt），节省了大量逆向工作。
- 明确指出存在的问题（如触摸屏优化差、bin文件名限制），避免读者走弯路。

**坑点（已标出）：**
1. `panel.bin` 文件名必须固定，否则设备树加载固件时会查找默认路径 `/lib/firmware/panel.bin`（或 config.txt 中指定的其他名称？文中未详细说明，但推测如此）。
2. 新内核无法安装头文件 → 强制降级内核（需注意树莓派启动流程中 `vmlinuz` 和 `initramfs` 的配置）。
3. 触摸屏驱动优化差，不建议用于生产环境。
